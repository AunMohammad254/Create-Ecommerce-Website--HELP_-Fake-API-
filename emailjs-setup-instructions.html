EmailJS Setup Instructions

This guide will help you set up EmailJS to enable real email sending functionality in your FakeStore application.

Note: The current implementation uses EmailJS, which allows sending emails directly from the frontend without a backend server. This is suitable for simple contact forms and notifications but has limitations for high-volume email sending.

Step 1: Create an EmailJS Account

  1. Go to https://www.emailjs.com/
  2. Click on "Create Free Account" or "Sign Up"
  3. Complete the registration process (you can use your Google account or email)

Step 2: Connect an Email Service

  1. After logging in, go to the "Email Services" tab in your dashboard
  2. Click "Add New Service"
  3. Choose an email service provider (Gmail, Outlook, or others)
  4. Follow the authentication process for your chosen provider
  5. Give your service a name (e.g., "FakeStore")
  6. Click "Create Service"
  7. Important: Note down your Service ID (it will look something like "service_xxxxxxx")

Step 3: Create an Email Template

  1. Go to the "Email Templates" tab in your dashboard
  2. Click "Create New Template"
  3. Give your template a name (e.g., "Order Confirmation")
  4. Set up the template with the following parameters:
    • To email: {{to_email}}
    • Subject: {{subject}}
    • Content: Include {{message}} in the body
  5. Click "Save"
  6. Important: Note down your Template ID (it will look something like "template_xxxxxxx")

Step 4: Get Your Public Key

  1. Go to the "Account" tab in your dashboard
  2. Find the "API Keys" section
  3. Important: Note down your Public Key (it will look something like "xxxxxxxxxxxxxxxxxx")

Step 5: Update Your Code

  1. Open the script.js file in your project
  2. Find the EmailJS initialization section (around line 698)
  3. Replace "YOUR_PUBLIC_KEY" with your actual Public Key
  4. Find the sendEmail function (around line 704)
  5. Replace "YOUR_SERVICE_ID" with your actual Service ID
  6. Replace "YOUR_TEMPLATE_ID" with your actual Template ID

Your code should look like this:

// Initialize EmailJS
(function() {
  // Replace with your EmailJS public key
  emailjs.init("your_actual_public_key");
})();

function sendEmail(to, subject, body) {
  // Prepare template parameters
  const templateParams = {
    to_email: to,
    subject: subject,
    message: body
  };
  
  // Send email using EmailJS
  // Replace with your service ID and template ID
  emailjs.send("your_actual_service_id", "your_actual_template_id", templateParams)
    .then(function(response) {
      // Success handling
    }, function(error) {
      // Error handling
    });
  
  return true;
}

Step 6: Test Your Implementation

  1. Save all your changes
  2. Reload your application
  3. Try to place an order or select a payment method that triggers an email
  4. Check your inbox for the email (it might be in spam/junk folder)

Troubleshooting

  • If emails are not being sent, check the browser console for error messages
  • Verify that your EmailJS service is properly connected
  • Make sure your template parameters match what's expected in your template
  • Check if you've reached the free tier limits (200 emails per month)
  • Some email providers might block automated emails, check your spam folder
